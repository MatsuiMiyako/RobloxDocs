
# 📘 Concept: Explorer Panel

## Definition
![[ExplorerUI.png]]
The Explorer panel is a window in Roblox Studio that shows the **hierarchy of all objects** in your game (like folders). It's how you see where everything is stored and how objects are parented.
## Core Purpose

The Explorer lets you:
- Navigate your game's object structure
- Parent and unparent objects (drag/drop)
- Quickly find where Scripts, Parts, GUIs, etc. are located
- See what runs on the **server**, the **client**, or on **startup**

Basically, it nicely organizes everything visually for you

## Key Properties / Methods

- You can **drag and drop** objects to move them
- Right-click to **Insert Object**, **Rename**, or **Delete**
- Shows **services** like `Workspace`, `ReplicatedStorage`, `StarterGui`, etc.
- Items appear in a tree structure (folders and children)

## When to Use It

- To make sure a `Script` is in the right location (e.g., `ServerScriptService`)
- To parent a UI object to `StarterGui`
- To delete broken or unnecessary parts/scripts
- To troubleshoot if something isn’t loading—check if it even exists

## Common Mistakes / Gotchas

- Forgetting to open it (`View > Explorer`)
- Accidentally deleting or dragging objects into the wrong parent
- Confusing where certain objects need to be (e.g., UI in `StarterGui`, not `Workspace`)
- Assuming something exists without verifying it in Explorer
