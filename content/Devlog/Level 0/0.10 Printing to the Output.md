# Concept: Printing to the Output

## Definition

The `print()` function in Lua sends a message to the **Output** window in Roblox Studio. It’s mainly used to:
- Test if your script runs
- Check the value of a variable
- Debug errors or behavior
- Understand what your code is doing
## Core Purpose

Use `print()` to:
- Prove that a line of code was executed
- Display the value of something at runtime
- Catch bugs before they become major problems
- Understand control flow (what runs, when, and why)

Think of it as your x-ray goggles—it shows what’s really happening under the hood.
## Key Properties / Methods

- Only visible if the **Output** window is open (View > Output)
- Only prints from running scripts
- Works in both `Script` and `LocalScript`
- Accepts text, numbers, booleans, tables, even functions

## When to Use It

- First line in any new script to make sure it runs
- Inside `if` statements to check which path is taken
- To debug player values, object names, and logic conditions
- Before and after an event (e.g., a button click)

## Common Mistakes / Gotchas

- Forgetting to open the Output window (View > Output)
- Assuming `print()` works in `ModuleScript` without being called
- Printing too much (can flood your output and hide bugs)
- Forgetting to remove `print()` lines after testing

## Code Example

```lua
print("Script has started")

local playerName = "RobloxUser123"
print("Player name is: " .. playerName) --This will output Player name is: RobloxUser123

local coins = 50
if coins >= 100 then
    print("Enough coins!")
else
    print("Not enough coins...")
end
-- Before adding anything too complicated to your code, you can start with print statements first to see if it actually works first
```
