
# Concept: Basic Parts and Models

## Definition
![[Pasted image 20250709173236.png]] ![[Pasted image 20250709173313.png]]

**Parts** are the simplest 3D building blocks in Roblox—like cubes, spheres, wedges—that make up the world. A **Model** is a container that groups multiple objects (like parts, scripts, and attachments) into a single movable structure.


## Core Purpose

- Use **Parts** to build the game world: floors, walls, buttons, objects, etc.
- Use **Models** to group related items so you can move them together, clone them, or manage them as one unit
- Every 3D object in Roblox is made of Parts, whether directly or inside a Model

## Key Properties / Methods

**Parts:**
- `Anchored`: Stops the part from falling
- `CanCollide`: Can the player walk through it?
- `Transparency`: Visibility of the part
- `Color`: Appearance
- `Size`: Dimensions in studs
- `Shape`: Can be block, sphere, wedge, cylinder

**Models:**
- Contain multiple objects
- Have no physical shape themselves (just a folder-like container)
- Often contain a `PrimaryPart` so you can move the whole model via code
## When to Use It

- Use **Parts** when creating physical objects like platforms, buttons, floors, items
- Use **Models** when building something with multiple parts (e.g., a house, NPC, trap)
- Use **Models** when you want to move, clone, or spawn complex structures

## Common Mistakes / Gotchas

- Forgetting to anchor parts—they fall when the game starts
- Using too many ungrouped parts—makes editing a mess
- Not setting `PrimaryPart` in models—can't move/position it via script
- Trying to change `Color` of a model directly—doesn’t work, you have to loop through parts
