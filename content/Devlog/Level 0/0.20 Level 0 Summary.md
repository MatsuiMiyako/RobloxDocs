# Concept: Level 0 Summary

## Definition

Level 0 introduced the absolute basics of Roblox Studio: how the interface works, how to insert and control parts, and how the game world behaves. This level focused more on understanding Studio, not too much coding logic.

## Core Purpose

To help brand-new users:
- Understand how Roblox Studio is structured
- Get comfortable navigating, inserting, and editing objects
- Learn the fundamental tools needed before writing full scripts

## Key Properties / Methods

- Explorer & Properties panels — control every object in the game
- Part types — like anchored, CanCollide, and models
- Parenting system — object hierarchy inside `game`
- Test modes — how to properly simulate your game
- Assets — `Toolbox` vs building your own and plugin usage

## When to Use It

- When opening Roblox Studio for the first time
- Before diving into scripts or systems
- When teaching others who have never touched Studio before

## Common Mistakes / Gotchas

- Confusing `Play` and `Run` (no player spawns in Run)
- Inserting Toolbox models without checking them
- Forgetting to anchor parts — funny things happen
- Using `LocalScripts` in the wrong services
- Not using Server/Client mode to test multiplayer logic

## Code Example

```lua
-- These were the types of things you learned to write in Level 0
-- a little scripting summary of Level 0 scripting concepts

-- Variable
local part = Instance.new("Part")

-- Property setup
part.Anchored = true
part.Position = Vector3.new(0, 10, 0)
part.Parent = workspace

-- Function
local function describePart(p)
	print("Anchored:", p.Anchored)
	print("Position:", p.Position)
end

-- If-statement
if part.Anchored == true then
	print("The part is anchored and won't fall.")
end

-- Loop (counting down from 3)
for i = 3, 1, -1 do
	print("Game starts in:", i)
	wait(1)
end

-- Call the function
describePart(part)
```
