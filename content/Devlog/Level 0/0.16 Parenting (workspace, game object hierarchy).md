# Concept: Parenting (workspace, game object hierarchy)

## Definition

Parenting in Roblox means placing one object inside another in the `Explorer` panel.  
This builds a hierarchy that determines how things behave in the game.

- `game` is the top-level container
- `workspace` is where visible objects live
- Every object has a `.Parent` that defines where it exists in the hierarchy

## Core Purpose

- Tells Roblox where the object lives in the game
- Affects visibility, physics, and scripting access
- Determines if an object is cloned, replicated, or deleted

## Key Properties / Methods

- `.Parent` — sets or gets the parent of an object
- `game.Workspace`, `game.ReplicatedStorage` — common parents
- `Explorer hierarchy` — visual way to parent objects

## When to Use It

- Putting a part into the `Workspace` to make it appear
- Storing models in `ReplicatedStorage` before cloning
- Organizing objects logically in the game

## Common Mistakes / Gotchas

- Forgetting to parent an object → it won’t show up or run
- Parenting GUI elements to `Workspace` (they won’t show)
- Putting scripts in the wrong service (e.g., `ServerScript` in `StarterGui`)

## Code Example

```lua
local part = Instance.new("Part")
part.Parent = workspace  -- Makes it visible in the game world
```
