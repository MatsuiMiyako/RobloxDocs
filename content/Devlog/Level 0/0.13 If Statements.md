
# Concept: If Statements

## Definition

An `if` statement checks whether a condition is true. If it is, it runs the code inside it. If not, it skips it (or runs something else using `else`).

```lua
if score >= 100 then
    print("You win!")
end
```

## Core Purpose

Use `if` statements to:
- Run code only when something is true
- Handle different outcomes (win/lose, pass/fail, alive/dead)
- Build logic into your scripts
- Control what happens and when
They’re the decision makers in your game’s brain.

## Syntax Structure

```lua
if condition then
    -- code that runs if true
end

-- optional else
if condition then
    -- code if true
else
    -- code if false
end
```

- Always use then after the condition
- Always close the block with end
- You can add elseif for more complex branching

## When to Use It

- Checking if a player has enough coins to buy something    
- Running code only if a condition is true
- Triggering game over when health <= 0
- Making choices inside events like touches or clicks

## Common Mistakes / Gotchas

- Forgetting `then` or `end` → script crashes    
- Using `=` instead of `==` for comparison (Lua uses `==` for checks)
- Comparing incompatible types (e.g., number vs. string)
- Writing too much logic inside a single `if` instead of breaking it up

## Code Example

```lua
local coins = 75

-- Basic if
if coins >= 50 then
    print("You can buy the item.")
end

-- If-else
if coins >= 100 then
    print("You're rich!")
else
    print("Keep grinding.")
end
```
