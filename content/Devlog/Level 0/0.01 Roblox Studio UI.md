# Concept: Roblox Studio UI

## Definition
![[HomeTabUI.png]]
The Roblox UI (inside Roblox Studio) is the interface where you create, modify, and organize your game. It includes panels like the **Explorer**, **Properties**, **Toolbox**, and **Viewport** which lets you visually place objects, set values, and access your game hierarchy.

## Core Purpose

The UI exists to let developers:
- Add and arrange objects (e.g. parts, models, scripts) visually
- Configure those objects through the **Properties panel**
- Navigate the object hierarchy in **Explorer**
- Test, edit, and manage the game without writing code all the time

Basically, it‚Äôs how you ‚Äúsee‚Äù and ‚Äútouch‚Äù your game outside of scripts.

## Key Properties / Methods

- **Explorer**: Shows your game's object tree (e.g., `Workspace > Part`)
- **Properties Panel**: Lets you change attributes like `Size`, `Color`, `Anchored`
- **Viewport**: The 3D space where you place and move things
- **Output**: Shows print statements and errors from your scripts
- **Toolbox**: Lets you insert models/assets from the Roblox library
## When to Use It

- When you‚Äôre adding a new Part or UI element
- To check if an object is parented correctly (e.g., is my script inside `StarterPlayerScripts`?)
- To change properties without writing code (e.g., make a part `Anchored`)
- To debug if something didn‚Äôt appear in-game

## Common Mistakes / Gotchas

- Not realizing the **Explorer** needs to be manually opened (`View > Explorer`)
- Confusing `StarterGui`, `PlayerGui`, and `ScreenGui` (UI may not show up where expected)
- Dragging parts into wrong containers (e.g., putting a Part in `StarterPack` by mistake)
- Thinking UI is visible in Studio but not realizing it only shows during **Play** mode
