# Concept: What is a Script

## Definition

A **Script** in Roblox is a Lua file that runs code on the server. It’s used to control game logic that needs to be secure, universal, or affect the whole game world—like spawning enemies, tracking scores, or handling damage.

## Core Purpose

Scripts:
- Run on the server (not the player’s device)
- Control things that should not be tampered with by players
- Manage shared game logic, systems, and data
- Communicate with clients using RemoteEvents

Think of Scripts as the brain of the game world—what’s true for everyone.

## Key Properties / Methods

- Scripts only run in **server-side containers**, such as:
	  - `ServerScriptService`
	  - `Workspace`
	  - `StarterCharacterScripts` (partially)
- Scripts won’t run if placed in `StarterGui` or `ReplicatedStorage`
- Use **RemoteEvents** to talk to LocalScripts on the client

## When to Use It

- Creating new parts, enemies, or effects in the world
- Damage systems, enemy AI, and wave spawning
- Server-side data storage and leaderboards
- Centralized control of game rules

## Common Mistakes / Gotchas

- Putting a Script inside `StarterGui` and wondering why it doesn't run
- Using a Script for things that should be client-side (like UI)
- Forgetting that **Scripts don’t have access to player UI**
- Confusing Scripts with LocalScripts or ModuleScripts

## Code Example

```lua
-- Script inside ServerScriptService

print("Server script is running!")

game.Players.PlayerAdded:Connect(function(player)
    print(player.Name .. " joined the game!")
end)
```
