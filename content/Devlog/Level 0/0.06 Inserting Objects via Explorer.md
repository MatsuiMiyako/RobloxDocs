
# Concept: Inserting Objects via Explorer

## Definition

In Roblox Studio, you can right-click on any object in the **Explorer** and select `Insert Object` to add a child object to it. This is the manual way to build your game’s object hierarchy without relying on scripts or dragging from the Toolbox.

## Core Purpose

Manually inserting objects:
- Gives you full control over where things are parented
- Helps avoid invisible UI, broken scripts, and misplaced assets
- Teaches how hierarchy affects game behavior
- Is required for setting up systems like GUIs, RemoteEvents, tools, and scripts correctly

## Key Properties / Methods

| Insert Into            | You’d Insert…                        |
| ---------------------- | ------------------------------------ |
| `Workspace`            | Parts, Models, Tools                 |
| `StarterGui`           | `ScreenGui` for UI                   |
| `ReplicatedStorage`    | RemoteEvents, shared Modules, assets |
| `ServerScriptService`  | Server-side Scripts                  |
| `StarterPlayerScripts` | Client-side LocalScripts             |
| `StarterPack`          | Tools that players spawn with        |

## When to Use It

- Setting up systems like leaderboards, tool inventories, or menus
- Organizing scripts or modules so they’re in the right container
- Creating new objects without writing code
- Structuring your project for clarity and logic
## Common Mistakes / Gotchas

- Putting UI elements directly into `Workspace` (they won’t show)
- Placing LocalScripts in the wrong spot (e.g., in `Workspace`)
- Not understanding the difference between `ReplicatedStorage` vs `ServerScriptService`
- Using Toolbox assets without checking where they inserted themselves


## Code Example

```lua
-- You can also parent objects via script:
local part = Instance.new("Part")
part.Size = Vector3.new(4,1,4)
part.Parent = game.Workspace
```
