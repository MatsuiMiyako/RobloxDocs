# Concept: Properties Panel

## Definition
![[PropertiesUI.png]]

The Properties panel in Roblox Studio displays all the editable attributes of the selected object—like `Size`, `Position`, `Color`, `Anchored`, `Transparency`, and more. It’s how you change the behavior or appearance of objects without writing code.
## Core Purpose

The Properties panel allows you to:
- Modify how objects look and behave
- Quickly configure objects without needing scripts
- Understand what settings each type of object exposes
- Debug if something seems off (e.g. part not anchored, collision disabled)

## Key Properties / Methods

- `Anchored` – stops parts from falling due to gravity
- `CanCollide` – lets players/objects walk through it or not
- `Transparency` – makes parts invisible or see-through
- `Size` – controls dimensions (in studs)
- `Color` – self-explanatory
- `Text` – for UI text labels/buttons
- `Visible` – whether UI shows up or not

## When to Use It

- When placing a part and needing it to stay still (`Anchored = true`)
- Making buttons invisible until certain conditions
- Tuning size/scale visually before scripting
- Debugging a part that’s “not working” (wrong property setting)

## Common Mistakes / Gotchas

- Forgetting to anchor parts—they fall during play
- Not realizing UI won’t show because `Visible` is unchecked
- Changing the wrong property (e.g., modifying a UI Frame instead of the TextButton inside)
- Thinking Properties auto-update in scripts—they don’t unless you set them in code

## Code Example

```lua
-- You can also set properties through script
local part = workspace.MyPart
part.Anchored = true
part.Transparency = 0.5
part.Color = Color3.fromRGB(255, 0, 0)
```
