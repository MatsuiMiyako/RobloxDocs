# Concept: Comments

## Definition

A **comment** is a line of text that the script completely ignores when running. It’s used to leave notes, label sections, or explain your logic. In Lua, single-line comments use `--` and multi-line comments use `--[["Everything in here will be ignored when running"]]`.

## Core Purpose

Comments are for:
- Explaining why you wrote something (not just what it does)
- Labeling sections of your code (setup, logic, cleanup)
- Making TODO notes while building
- Helping teammates (or future-you) understand the script instantly

Good code is read more than it's written. Comments make it readable.

## Comment Syntax

```lua
-- This is a single-line comment

--[[ 
This is a multi-line comment.
Great for explaining a section or idea.
]]

local coins = 50  -- This variable stores the player's coins
```


## When to Use It

- Before writing a function: describe what it’s supposed to do
- At the top of a script: give an overview of its purpose
- When doing something non-obvious (like using a weird workaround)
- To mark areas you’ll improve or expand later (`-- TODO: Add boss logic`)
## Common Mistakes / Gotchas

- Writing obvious comments (e.g., `-- Adds 1 to score` right above `score = score + 1`)
- Forgetting to update comments when code changes
- Using comments as a crutch instead of writing clear code
- Leaving TODOs forever—use them, but clean them up

## Code Example

```lua
-- -- Handles damage when player touches enemy
local damage = 10

--[[ 
If the player's health is above zero,
apply damage and update the UI.
]]
function takeDamage(player)
    player.Health = player.Health - damage
end

-- TODO: Add knockback effect
```
