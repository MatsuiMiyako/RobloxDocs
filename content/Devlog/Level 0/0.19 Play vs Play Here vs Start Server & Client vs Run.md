# Concept: Play vs Play Here vs Start Server & Client vs Run

## Definition

Roblox Studio has multiple ways to test your game, each running a different part of the simulation:

- **Play**: Starts the game with a local player from the spawn point
- **Play Here**: Spawns the player at the current camera location
- **Start Server/Client**: Simulates real multiplayer — server and clients
- **Run**: Starts the game logic without spawning a player

## Core Purpose

They allow you to test your game in controlled ways — from simple single-player testing to full multiplayer simulation with real server/client behavior. Understanding the difference is crucial for debugging, especially with scripts

## Key Properties / Methods

- Play: Spawns your player at spawn
- Play Here: Spawns player at camera position
- Start Server: Simulates full server
- Start Client: Simulates multiple players connecting
- Run: Simulates game without players

## When to Use It

- Use **Play** to test basic gameplay, UI, camera, and movement
- Use **Play Here** when you want to test a location far from spawn
- Use **Start Server & Client** to test RemoteEvents, filtering, multiplayer logic
- Use **Run** to check server scripts, environment behavior, or animations without player interference
## Common Mistakes / Gotchas

- Not testing server-side code with Start Server
- Only testing with Play and assuming everything works in multiplayer
- Expecting server scripts to run in Play mode (they don’t always)

## Code Example

```lua
-- ServerScript
print("This is server-side")

-- LocalScript
print("This is client-side")
```
